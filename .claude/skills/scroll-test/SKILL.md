# Автотесты скролл-эффектов (headless Edge + CDP)

Скрипты лежат в `tests/`. Требования: `python -m pip install --user websocket-client`,
Edge по пути `C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe`.

⚠️ Обычный `msedge --screenshot=файл` на этой машине блокируется системой
(«Отказано в доступе»), поэтому все скриншоты снимаются через CDP
`Page.captureScreenshot` — так и задумано в скриптах.

## Подготовка

- Для локального прогона поднять сервер: `python -m http.server 8010 --directory <корень проекта>`
  (в фоне), после тестов остановить.
- Перед КАЖДЫМ прогоном удалять профиль headless-браузера (иначе возьмёт
  страницу из кеша и покажет старое поведение): папки `cdpprof*` рядом со скриптом.

## Тесты (URL — первый аргумент; без него берут живой сайт)

- `python tests/fastscroll.py [url]` — резкий скролл через пин героя.
  Ожидаемо: `RESULT: no flash detected`. Если `REPRODUCED` — затемнение
  не успевает за скроллом (рывок полупрозрачного видео при отпине).
- `python tests/scrolltest.py [url]` — медленная прокрутка, печатает
  scrollY / opacity затемнения / transform видео и сохраняет кадры
  `f*.jpg` рядом со скриптом — смотреть глазами композицию перехода.
- `python tests/reloadtest.py [url]` — поведение F5: после перезагрузки
  scrollY должен быть 0 и якорь убран; переход по ссылке с якорем
  (`#contact`) должен по-прежнему прокручивать к якорю.
- `python tests/renderfav.py` — рендер favicon.svg в прозрачный PNG 512px
  (нужен локальный сервер на :8010).

## Тонкие места, которые эти тесты охраняют

- Пин героя: `scrub: 0.3`, затемнение `.hero__fadeout` завершается к ~85%
  дистанции пина — раньше нельзя (длинная чернота), позже нельзя (рывок).
- F5 всегда открывает верх: `history.scrollRestoration='manual'` +
  `ScrollTrigger.clearScrollMemory('manual')` + сброс после refresh.
- Лента проектов едет через `scrollLeft` (НЕ transform-анимацией — гигантский
  слой превышает лимит GPU-текстуры на телефонах с DPR 3 и мигает).
